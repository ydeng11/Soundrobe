## Overview

Auto Tagger v1.0 delivers a Python CLI for Navidrome-oriented audio metadata automation. The milestone established the project architecture, core tag read/write support, Beets/MusicBrainz lookup, OpenRouter-assisted candidate selection and fallback generation, quality validation, Navidrome features, batch/interactive command surfaces, and release packaging artifacts.

## Milestone Summary

**Key Decisions:**
- Use a Python `src/` layout with Click, Rich, Pydantic Settings, and hatchling packaging.
- Use mutagen as the audio metadata abstraction for MP3, FLAC/Vorbis, and MP4/M4A.
- Isolate Beets, LLM, quality validation, feature enrichment, and workflow orchestration into separate package boundaries.
- Keep external tools and network services behind injectable boundaries for deterministic tests.
- Treat real PyPI/Homebrew publishing as a credentialed release action, not an automatic local task.

**Issues Resolved:**
- Basic CLI and configuration system implemented.
- Multi-format metadata read/write and Navidrome multi-artist tags implemented.
- Beets lookup, folder fallback, and match caching implemented.
- LLM selection/fallback with cost tracking implemented.
- Audio/LRC/metadata health checks and reporting implemented.
- Cover art, lyrics, compilation, batch, interactive, and distribution artifacts implemented.

**Issues Deferred:**
- REQ-CT-004 additional artist fields (`composer`, `lyricist`, `conductor`) deferred to v2.
- Real PyPI upload and Homebrew SHA update deferred to manual release.

**Technical Debt:**
- Apply/write workflows remain intentionally conservative and should be expanded with more real-media integration tests before broad unattended use.
- Homebrew formula is a template until a published source archive and checksum exist.
