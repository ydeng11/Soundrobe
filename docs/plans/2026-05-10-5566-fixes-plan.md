# Fix 5566 Library Support Plan

> **Goal:** Two fixes — `.wav` audio support and smart folder-name cleanup for artist/album extraction.

---

## Problem 1: WAV files not detected (3 of 4 albums invisible)

**Root cause:** `.wav` is not in `SUPPORTED_EXTENSIONS`. The 3 albums `2003-04《挚爱》`, `2005-08《好久不见》`, `2008-01《喝采》` are entirely `.wav` files.

**Fix:**
1. Add `AudioFormat.WAV` enum value
2. Add `".wav": AudioFormat.WAV` to `SUPPORTED_EXTENSIONS`
3. WAV files use ID3 tags (same read/write path as MP3) — reuse `_read_mp3_tags` / `_write_mp3_tags` or create thin wrappers
4. Add WAV fixture to the factory
5. Tests: read/write round-trip, format detection, file discovery

---

## Problem 2: Folder names mismatch with MusicBrainz/Spotify

**Root cause:** `parse_album_path()` returns the raw folder name like `2003-04《挚爱》`, but MusicBrainz knows this album as just `挚爱` (or `摯愛`).

**Fix:**
1. Add `clean_folder_name(name: str) -> str` to `fallback.py`:
   - Strip leading date prefix: `^\d{4}-\d{2}` → removes `2003-04`
   - Strip Chinese bookmarks: `《》` `「」` `【】`
   - Strip leading/trailing whitespace and separators
2. Use cleaned name for `artist_hint` and `album_hint` in lookups
3. Keep original folder path for display/writing — only hints are cleaned

**Before:** `parse_album_path("2003-04《挚爱》")` → `album_hint="2003-04《挚爱》"`
**After:** → `album_hint="挚爱"`
